# ADR-0013 — Internacionalização (Inglês + Português) desde o MVP

- **Status:** Aceito
- **Data:** 2026-06-05
- **Decisores:** PM, UX/UI, Tech Lead

## Contexto
Decisão do stakeholder: o MVP suporta **Inglês e Português**, com **localização (i18n) configurada
desde o início**. Adicionar i18n depois é caro (strings espalhadas, formatação hardcoded); fazer
desde já é barato se a fundação existir.

## Decisão
- Usar **`Microsoft.Extensions.Localization`** (no shared framework do ASP.NET Core — sem pacote
  extra) com **`IStringLocalizer`** e arquivos **`.resx`**:
  - `Resources/SharedResource.resx` (neutro / **Inglês**, fallback)
  - `Resources/SharedResource.pt-BR.resx` (**Português**)
- **`RequestLocalization`** com culturas suportadas `en` e `pt-BR`; **padrão `pt-BR`** (público
  atual). Seleção de cultura por usuário entra como feature.
- **`InvariantGlobalization` desligado** (precisamos de ICU). No container, usar imagem **com ICU**
  (Debian, não a chiseled/sem-ICU). Ver [`Dockerfile`](../../Dockerfile).
- Regra de ouro: **nenhuma string de UI hardcoded** — tudo via localizer.

## Consequências
- ✅ Adicionar um terceiro idioma no futuro = novo `.resx`, sem refatorar telas.
- ✅ Formatação de datas/números respeita a cultura.
- ⚠️ Imagem de container um pouco maior (ICU) — irrelevante no Pi.
- ⚠️ Disciplina de equipe: revisar PRs para barrar strings hardcoded.
